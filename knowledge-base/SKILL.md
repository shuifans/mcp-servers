# 企业知识助手 (knowledge-base MCP)

搜索本地 Mac 文档库 + 阿里内网(ATA/云知道) + 优质公网，返回带出处的精准结果。

## 何时使用

当用户：
- 需要查找工作资料、技术方案、行业报告
- 询问公司内部实践（ATA/云知道）
- 需要对比内部做法与外部最佳实践
- 提到"搜索"、"找资料"、"有没有关于...的文档"

## 使用策略

1. **分析问题** → 判断需要搜索哪些来源：
   - 内部实践/公司经验 → search_ata + search_yunzhidao
   - 本地积累的文档 → search_local
   - 行业最佳实践/英文资料 → search_public
   - 综合性问题 → 多来源并行搜索

2. **执行搜索** → 优先并行调用多个搜索工具：
   - 中文关键词搜本地/ATA/云知道
   - 英文关键词搜公网
   - 概念型问题拆成多个具体子查询分别搜索

3. **归纳回答** → 基于搜索结果：
   - 每个关键判断附引用（来源路径或URL）
   - 区分：📋 资料直接结论 / 🔗 跨来源归纳 / ⚠️ 信息缺口
   - 部分证据时给出有限结论，标注缺口
   - 只有完全无证据时才说"未找到"

## 可用工具

| 工具 | 用途 | 适合场景 |
|------|------|---------|
| search_local | 搜索本地已索引文档 | 找自己积累的PPT/PDF/文档 |
| search_ata | 搜索 ATA 内部技术文档 | 找公司内部技术方案和经验 |
| search_yunzhidao | 搜索云知道知识库 | 找内部流程和知识文档 |
| search_public | 搜索公网优质来源 | 找行业最佳实践和权威资料 |
| internal_login | SSO 登录 | ATA/云知道搜索前需要 |
| scan_index | 触发全量索引扫描 | 添加新目录后更新索引 |
| manage_directory | 管理授权目录 | 添加/移除/列出索引目录 |
| index_status | 查看索引状态 | 了解当前索引了多少文档 |

## 注意事项

- ATA 和云知道需要先 SSO 登录（调用 internal_login）
- 公网搜索用英文关键词效果更好
- 搜索结果已自动过滤低质量来源
- 本地搜索基于全文匹配，用核心关键词效果更好
